# AI Alarm System — 비즈니스 가이드

## 시스템 개요

AI Alarm System은 제조 현장의 HMI(Human-Machine Interface) 패널 화면을 AI가 자동으로 분석하여, 장비 이상 여부를 실시간으로 판단하고 알림을 제공하는 시스템입니다.

```
┌─────────────┐      ┌─────────────┐      ┌──────────────────┐
│  HMI 패널    │ ──→  │  분석 서버   │ ──→  │  Azure OpenAI    │
│  화면 캡처   │      │  (자동 판단)  │      │  (AI 이미지 분석) │
└─────────────┘      └──────┬──────┘      └──────────────────┘
                            │
                    ┌───────┴───────┐
                    │               │
              ✅ 정상(OK)     ❌ 이상(NG)
                              │
                        📧 이메일 알림
                        📋 로그 기록
```

## 분석 대상 장비

하나의 HMI 화면에서 4개 장비를 동시에 모니터링합니다:

| 장비 | 설명 | 모니터링 항목 |
|------|------|-------------|
| **S520** | Preheating & Curing (예열/경화) | Curing Oven + Preheating Oven 수량 28개 |
| **S530** | Cooling (냉각) | Cooling 1 Line + Cooling 2 Line 수량 28개 |
| **S540** | Robot (로봇) | 1~6번 스테이션 상태 (색상 + 수량) |
| **S810** | Housing Cooling (하우징 냉각) | Cooling 1 line + Cooling 2 line 수량 30개 |

## 판단 기준

### ✅ OK (정상)
- 모든 장비가 정상 식별됨
- S520, S530, S810: 모든 수량이 3,000 미만
- S540: 모든 스테이션 배경색이 정상 (초록/흰색)

### ❌ NG (이상)
- S520, S530, S810: **수량이 3,000 이상**인 항목 발견
- S540: 스테이션 배경색이 **빨간색** 또는 **검은색**

### ⚠️ UNKNOWN (판단 불가)
- 4개 장비 중 하나라도 화면에서 식별 불가
- 필요한 데이터를 모두 읽어오지 못한 경우
- → **이메일 알림 발송** (담당자 확인 필요)

## 알림 체계

| 상황 | 동작 |
|------|------|
| OK | 결과 기록, Client 화면에 초록색 표시 |
| NG | 결과 기록, Client 화면에 빨간색 표시, NG 항목 상세 표시 |
| UNKNOWN | 결과 기록 + **이메일 알림** + 원본 이미지 별도 저장 |
| TIMEOUT | 결과 기록, Client 화면에 회색 표시 (AI 응답 지연) |

## 모니터링 방식

- **단건 분석**: 이미지 하나를 선택하여 즉시 분석
- **주기적 분석**: 5초 또는 10초 간격으로 자동 반복 분석
- **응답 시간 추적**: 각 분석의 처리 시간(ms)을 기록하여 지연 감지

## 데이터 보관

모든 분석 결과는 자동으로 저장됩니다:

| 항목 | 저장 위치 | 형식 |
|------|----------|------|
| 판단 결과 | `data/results/` | JSON (요청별 1파일) |
| 분석 로그 | `data/logs/` | 텍스트 (일별 1파일) |
| UNKNOWN 이미지 | `data/unknown_images/` | 원본 이미지 복사본 |

## 테스트 이미지 관리

테스트용 이미지는 기대 결과별로 폴더에 분류합니다:

```
test_images/
├── ok/        ← 정상으로 판단되어야 하는 이미지
├── ng/        ← 이상으로 판단되어야 하는 이미지
└── unknown/   ← 판단 불가 이미지
```

Client GUI에서 폴더를 선택하면 AI 판단 결과와 기대값을 자동 비교하여 정확도를 확인할 수 있습니다.

## 판단 조건 변경

판단 조건은 코드 수정 없이 설정 파일(`server/prompt_config.yaml`)에서 변경할 수 있습니다:
- NG 임계값 변경 (예: 3,000 → 2,500)
- 새로운 장비 추가
- 판단 조건 추가/수정

## 향후 확장 가능 사항

- 실시간 화면 캡처 연동 (RPA 자동화)
- 대시보드 웹 UI
- 판단 이력 통계/트렌드 분석
- 추가 장비/조건 확장
- 알림 채널 다양화 (Teams, Slack 등)
