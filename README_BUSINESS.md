# AI Alarm System — 비즈니스 가이드

## 시스템 개요

AI Alarm System은 제조 현장의 HMI(Human-Machine Interface) 패널 화면을 AI가 자동으로 분석하여, 장비 이상 여부를 실시간으로 판단하고 알림을 제공하는 시스템입니다.

```
┌─────────────┐      ┌──────────────────────────────────────┐
│  HMI 패널    │      │              분석 서버                │
│  화면 캡처   │ ──→  │                                      │
└─────────────┘      │  4개 패널 크롭 (1회)                  │
                     │         │                            │
                     │    ┌────┴────┐                       │
                     │    ▼         ▼                       │
                     │  [DI]      [LLM]                     │
                     │  숫자 추출  색상 감지                  │
                     │  (병렬)    (병렬)                     │
                     │    │         │                       │
                     │    └────┬────┘                       │
                     │         ▼                            │
                     │    최종 판정 병합                      │
                     └──────────┬───────────────────────────┘
                                │
                    ┌───────────┼───────────┐
                    ▼           ▼           ▼
              ✅ 정상(OK)  ❌ 이상(NG)  ⚠️ 불명(UNKNOWN)
                                              │
                                        📧 이메일 알림
                                        📋 로그 기록
```

## 분석 파이프라인

### 1단계: 패널 크롭 (1회)
전체 HMI 이미지(1920×1170)를 4개 패널로 분할합니다. 크롭된 이미지는 DI와 LLM이 공유합니다.

### 2단계: 병렬 처리

| 처리 | 담당 | 대상 | 결과 |
|------|------|------|------|
| 숫자 추출 | Azure Document Intelligence | S520, S530, S810 | 테이블 수치값 |
| 색상 감지 | Azure OpenAI GPT-4o Vision | S520, S530, S540, S810 | 빨간 영역 감지 |

두 처리는 **동시에** 실행되어 전체 처리 시간을 최소화합니다 (약 7~9초).

### 3단계: DI 검증
DI 추출 결과에서 장비 ID 누락 또는 값 개수 부족 시:
- LLM 호출 없이 즉시 **UNKNOWN** 반환
- 추출된 부분 데이터를 로그에 기록
- 이메일 알림 발송

### 4단계: 최종 판정 병합
- 숫자 NG: DI 추출값 중 3,000 이상인 항목
- 색상 NG: LLM이 감지한 빨간 배경 영역
- 두 결과를 합산하여 최종 OK/NG/UNKNOWN 결정

## 분석 대상 장비

하나의 HMI 화면에서 4개 장비를 동시에 모니터링합니다:

| 장비 | 설명 | 숫자 추출 | 색상 감지 |
|------|------|----------|----------|
| **S520** | Preheating & Curing (예열/경화) | 28개 값 (DI) | 빨간 영역 (LLM) |
| **S530** | Cooling (냉각) | 28개 값 (DI) | 빨간 영역 (LLM) |
| **S540** | Robot (로봇) | 없음 | 12개 스테이션 색상 (LLM) |
| **S810** | Housing Cooling (하우징 냉각) | 30개 값 (DI) | 빨간 영역 (LLM) |

## 판단 기준

### ✅ OK (정상)
- 모든 장비가 정상 식별됨
- S520, S530, S810: 모든 수량이 3,000 미만
- S520, S530, S810: 빨간 배경 영역 없음
- S540: 모든 스테이션 배경색이 정상 (초록/흰색)

### ❌ NG (이상)
- S520, S530, S810: **수량이 3,000 이상**인 항목 발견
- S520, S530, S810: **빨간 배경 영역** 감지 (텍스트 포함 기록)
- S540: 스테이션 배경색이 **빨간색** 또는 **검은색**

### ⚠️ UNKNOWN (판단 불가)
- 4개 장비 중 하나라도 화면에서 식별 불가
- 필요한 데이터 값 개수 부족 (화면 가림/조작 의심)
- → **이메일 알림 발송** + 부분 추출 데이터 로그 기록

## 알림 체계

| 상황 | 동작 |
|------|------|
| OK | 로그 기록, Client 화면에 초록색 표시 |
| NG | 로그 기록, Client 화면에 빨간색 표시, NG 항목 상세 표시 |
| UNKNOWN | 로그 기록 + **이메일 알림** + 부분 추출 데이터 기록 |
| TIMEOUT | 로그 기록, Client 화면에 회색 표시 (AI 응답 지연) |

> 이메일 알림은 `SNS_ENABLED=true` 설정 시에만 발송됩니다.

## 모니터링 방식

- **단건 분석**: 이미지 하나를 선택하여 즉시 분석
- **주기적 분석**: 5초 또는 10초 간격으로 자동 반복 분석
- **응답 시간 추적**: 각 분석의 처리 시간(ms)을 기록하여 지연 감지

## 데이터 보관

| 항목 | 저장 위치 | 형식 | 저장 조건 |
|------|----------|------|----------|
| 판단 결과 | `data/results/` | JSON (요청별 1파일) | debug 모드에서만 |
| 분석 로그 | `data/logs/` | 텍스트 (일별 1파일) | 항상 |
| UNKNOWN 이미지 | `data/unknown_images/` | 원본 이미지 복사본 | debug 모드에서만 |

> 운영 환경(`python -m server.main`)에서는 로그만 기록되고 파일 저장은 비활성화됩니다.

## 판단 조건 변경

판단 조건은 코드 수정 없이 설정 파일에서 변경할 수 있습니다:

- `server/prompt_config.yaml` — LLM 색상 감지 프롬프트, 장비 정의
- `server/.env` — 임계값, 알림 활성화 여부, 모델 선택

## 향후 확장 가능 사항

- 실시간 화면 캡처 연동 (RPA 자동화)
- 대시보드 웹 UI
- 판단 이력 통계/트렌드 분석
- 추가 장비/조건 확장
- 알림 채널 다양화 (Teams, Slack 등)
